Use the `dotcms-github-issues` skill in **CREATE mode** to create a new GitHub issue in `dotCMS/core`.

User's request: $ARGUMENTS

**Quick draft:** If the user wants to create an issue quickly without going through the Issue Refinement step, they can say so explicitly — e.g. "draft", "rápido", "quick draft", "sin detalles completos", "sin resolver ambigüedades", "crear rápido". In that case, the skill skips Step 7b (Issue Refinement) and builds the issue body with a short placeholder for acceptance criteria (to be refined later).
