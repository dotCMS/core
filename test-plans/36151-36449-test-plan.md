# Test Plan: Issue #36151 — New Site/Folder Selector Field

**Issue:** [#36151](https://github.com/dotCMS/core/issues/36151)  
**PR:** [#36449](https://github.com/dotCMS/core/pull/36449)  
**Date:** 2026-07-07

---

## 1. Summary

This test plan covers the new popover-based Host/Folder field for Edit Contentlet, replacing the legacy `p-treeselect` with a two-panel overlay (sites + paginated folders tree), search, load-more pagination per level, staged selection with Select button, and copy-to-clipboard.

**Scope:** Unit (Jest/Spectator), E2E (Playwright), manual QA for visual/icon checks.

---

## 2. Feature Overview

| Area | New behavior |
|---|---|
| UI | Button trigger + `p-popover` with sites scroller (left) and folders tree (right) |
| Pagination | Load more sentinel per tree level when >40 items; appends next page |
| Search | Min 3 chars; backend `/api/v1/folder/search` with recursive flag |
| Selection | Staged in overlay; committed on Select; discarded on close |
| Display | Trigger shows `hostname / folder1 / folder2`; form value `hostname:path` |
| Copy | CDK Clipboard + check icon feedback |

## 6. Edge Cases

- Empty `folderPath` → default site (prefill + file-asset specs)
- Invalid folderPath → field functional, non-empty label (`host-folder-field-nested.spec.ts`)
- ≤40 folders → no load-more button (not E2E covered; unit partial)
- Search <3 chars → no API call (unit covered)

---

## 7. Out of Scope

- Postman/integration for `/api/v1/folder/search` (endpoint reused)
- Auto-scroll to selected folder on overlay open
- Visual truncation of long folder names
- Cross-site search E2E (starter env limitation)
